Features

HBA Datentransfer Übersicht

Um den HBA - Datentransfer bundeslandübergreifend zu digitalisieren, stellt die sberg it-systeme GmbH ein neues Modul für die elBook Plattform bereit. Damit können die nötigen Daten des HBA's digital übermittelt werden. Der absendende Kartenherausgeber kann die Daten des HBAs auf verschiedenen Wegen der elBook Plattform mitteilen und den empfangenen Kartenherausgeber festlegen, der von der elBook-Plattform per Mail über neue Daten informiert wird. Diese neuen Datensätze können dann direkt bei elBook abgeholt werden. Nur der Absender der HBA-Daten benötigt eine Lizenz für elBook. Der Empfänger kann die Daten über einen Resourcenlink (eine URL) ohne Lizenz abrufen.

HBA Datentransfer erstellen

Der absendende Kartenherausgeber des Datentransfers erstellt einen HBA-Transfer mit allen nötigen Angaben. Ein Datentransfer kann manuell über ein Formular oder per REST-API an elBook übergeben werden. Beim Eingang werden die Daten auf Vollständigkeit geprüft. Per API können auch mehrere Daten in einen Request übergeben werden. Alle Datentransferanträge werden abgelegt und gespeichert und nach einer bestimmten Zeit aus dem System gelöscht.

HBA Datentransfer erhalten

elBook erstellt aus den Datentransferanträgen einen Resourcenlink und einen Abholcode, verpackt alle Angaben in eine E-Mail und sendet sie an die Empfänger-Adresse, die mit den Anträgen übergeben wurde. Der Resourcenlink hat eine begrenzte Verfügbarkeit. Wird er geöffnet, wird man auf die elBook Plattform geleitet, die den Code abfragt. Nach der Verifizierung werden die Daten angezeigt und der Empfänger kann die Daten herunterladen.

UseCases (Fachliche Szenarien)

Im folgenden werden UseCases beschrieben die eine Übermittlung der HBA-Daten nötig machen und die durch den Einsatz von elBook im Alltag effizienter durchgeführt werden können.

UseCase: Umzug eines HBA Besitzers in ein anderes Bundesland

Zieht ein HBA Besitzer in ein anderes Bundesland, müssen die HBA-Daten an den neuen Kartenherausgeber übermittelt werden. Der HBA Besitzer bittet den aktuellen Herausgeber, die HBA-Daten an den neuen Kartenherausgeber zu übermitteln.

UseCase: HBA-Kartenherausgeber befindet sich nicht im Bundesland der SMC-B Beantragung

Wird eine SMC-B beantragt, muss bereits ein HBA vorhanden sein. Ist der Betriebsort für die SMC-B nicht im gleichen Bundesland kann ggf. kein HBA für diese Person gefunden werden. In diesem Fall müssen die HBA-Daten vom Kartenherausgeber des HBA's zum Beantragungsort der SMC-B übermittelt werden, so dass der Kartenherausgeber für die SMC-B die Daten abgleichen kann. Auch hier kann der HBA Besitzer den Kartenherausgeber des HBA's veranlassen die HBA-Daten an den Kartenherausgeber der SMC-B zu übermitteln.

UseCase: HBA Datensynchronisation zwischen HBA Kartenherausgebern (Ärztekammern) und SMC-B Kartenherausgeber (KVen)

Für Ärzte ist der HBA-Kartenherausgeber die Ärztekammer. Die SMC-B's werden wiederum durch die kassenärztlichen Vereinigungen herausgegeben. Da für eine SMC-B Beantragung der HBA nötig ist, kann es hier bei den KVen zu Problemen führen, wenn die HBA-Daten nicht zur Verfügung stehen. Diese Lücke wird durch elBook geschlossen. Die Ärztekammern können über elBook HBA-Daten an die Kartenherausgeber für die SMC-B (kassenärztliche Vereinigungen) senden. Somit wird die Informationslücke der KV für die Beantragung der SMC-B geschlossen. elBook ermöglicht das Senden von HBA-Daten auch gesammelt, womit die Ärztekammer eine große Anzahl von HBA-Daten übertragen kann.
